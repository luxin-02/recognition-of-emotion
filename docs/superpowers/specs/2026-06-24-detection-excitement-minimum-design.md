# Detection 兴奋指数最低分设计

## 目标

实际检测产生结果后，右上角图表中的“兴奋指数”不得低于 5 分，并将同一数值发送到报告。页面首次进入以及报告发送成功后的重置状态仍保持 0 分。

## 方案

仅修改 `src/utils/emotionCalculation.js` 中 `getRightTopValueData` 的无匹配数据兜底分支。当当前指标为“兴奋指数”，且识别结果中没有“兴奋、快乐、幸福”等可用于计算的情绪时，返回 5 分；其他右上角指标仍返回 0 分。

识别结果存在匹配项时继续使用现有 `formatRightTopScore` 公式，该公式对兴奋指数的基础分为 30，天然高于最低值。计算后的 `rightTopValueData` 同时供图表展示和 `addSmileAssess` 报告参数使用，因此无需在页面层重复处理。

## 验证标准

- 无匹配情绪时，检测结果中的兴奋指数为 5 分。
- 有匹配情绪时，兴奋指数继续使用现有公式。
- 图表展示值和报告参数值一致。
- 初始状态和报告发送成功后的重置状态仍为 0 分。
- 项目生产构建通过。

# 音乐页默认播放第一首设计

## 目标

用户刚进入音乐主题页面、尚未选择或播放歌曲时，点击播放/暂停控制按钮，应播放当前分类、当前搜索结果、当前页列表中的第一首歌曲。

## 方案

仅调整音乐页现有 `playMusic` 方法：

- 如果已有当前歌曲，维持原有暂停和继续播放行为。
- 如果没有当前歌曲且当前列表非空，调用现有 `musicBtn` 方法选择索引为 `0` 的歌曲。该方法继续负责设置当前歌曲、列表高亮、音频加载和播放状态。
- 如果当前列表为空，保持不播放，不显示额外错误。

不在列表加载完成后自动选择歌曲，以保留页面初始的“暂无当前歌曲”状态；不新增只使用一次的抽象。

## 影响范围

只修改 `src/views/front/views/music/index.vue` 中的播放/暂停入口，不调整接口、分页、搜索、分类切换、上一首或下一首逻辑。

## 验收标准

1. 页面首次加载且列表有歌曲时，点击播放/暂停按钮会播放列表第一首。
2. 列表为空时点击按钮不会报错或触发播放。
3. 已选择歌曲时，按钮仍能正常暂停和继续播放。
4. 项目构建通过，且没有无关文件改动。

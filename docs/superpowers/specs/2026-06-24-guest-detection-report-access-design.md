# 游客检测与报告入口权限设计

## 目标

限制游客执行以下操作：

- 在情绪检测页面点击“开始检测”。
- 在首页点击“报告中心”。

游客点击时使用项目现有消息提示显示“游客暂无权限”，并停留在当前页面。普通用户行为保持不变。

## 设计

在两个现有点击处理位置分别使用 Vuex 的 `isGuest` getter 判断身份：

- `src/views/front/views/detection/components/middle.vue`：在 `startCheck()` 开头判断。游客直接提示并返回，不调用摄像头相关逻辑。
- `src/views/front/index/Home.vue`：将报告中心的内联跳转改为页面方法。游客直接提示并返回，其他用户继续跳转 `/report`。

本次不禁用或隐藏按钮，不增加全局路由守卫，也不限制游客进入情绪检测页面或直接访问其他路由。

## 验证

- 游客点击“开始检测”后出现“游客暂无权限”，摄像头和检测流程不启动。
- 游客点击首页“报告中心”后出现“游客暂无权限”，路由不跳转。
- 普通用户点击两个入口时保持原有行为。
- 项目生产构建通过。
